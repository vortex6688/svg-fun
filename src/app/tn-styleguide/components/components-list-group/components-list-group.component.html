List group

List groups are a flexible and powerful component for displaying a series of content. List group items can be modified and extended to support just about any content within. They can also be used as navigation with the right modifier class.

Links and buttons

Use <a>s or <button>s to create actionable list group items with hover, disabled, and active states by adding .list-group-item-action. We separate these pseudo-classes to ensure list groups made of non-interactive elements (like <li>s or <div>s) donâ€™t provide a click or tap affordance.

Be sure to not use the standard .btn classes here.

The first group item The second group item The third group item The 4th group item The lest item in this group 
<div class="list-group">
  <a href="#" class="list-group-item active">
    The first group item
  </a>
  <a href="#" class="list-group-item list-group-item-action">The second group item</a>
  <a href="#" class="list-group-item list-group-item-action">The third group item</a>
  <a href="#" class="list-group-item list-group-item-action">The 4th group item</a>
  <a href="#" class="list-group-item list-group-item-action disabled">The lest item in this group</a>
</div>

With <button>s, you can also make use of the disabled attribute instead of the .disabled class. Sadly, <a>s donâ€™t support the disabled attribute.

<div class="list-group">
  <button type="button" class="list-group-item list-group-item-action active">
    The first group item
  </button>
  <button type="button" class="list-group-item list-group-item-action">The second group item</button>
  <button type="button" class="list-group-item list-group-item-action">The third group item</button>
  <button type="button" class="list-group-item list-group-item-action">The 4th group item</button>
  <button type="button" class="list-group-item list-group-item-action" disabled>The lest item in this group</button>
</div>

Conveying meaning to assistive technologies

Using color to add meaning only provides a visual indication, which will not be conveyed to users of assistive technologies â€“ such as screen readers. Ensure that information denoted by the color is either obvious from the content itself (e.g. the visible text), or is included through alternative means, such as additional text hidden with the .sr-only class.
